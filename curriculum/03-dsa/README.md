## The principle of DSA prep

Most students fail DSA because they:
- Watch hours of YouTube without coding.
- Jump topics randomly.
- Solve too many easy problems and never level up.
- Try to memorise solutions instead of recognising patterns.

The fix is simple:

```
1 concept → 5 easy problems → 5 medium problems → 1 hard problem
            ↑                  ↑                    ↑
       solve same day      next 2 days         end of the week
```

If a topic takes you 4-5 days at this pace, that's perfectly normal.

## Languages

- **Recommended:** Python (less syntax overhead, focus on logic).
- **Alternative:** Java or C++ if you want OOP/STL practice for product-company interviews.
- Pick one and **don't switch** mid-prep.

## Daily DSA targets

| Phase           | Problems/day | Topics in flight              |
| --------------- | ------------ | ----------------------------- |
| Weeks  1- 6     | 2 (easy)     | Arrays, strings, basics       |
| Weeks  7-12     | 3 (1E+2M)    | Linked List, Stack, Queue, Hash, Recursion |
| Weeks 13-18     | 3 (1M+1M+1H or 2M+1H) | Trees, Graphs, DP basics |
| Weeks 19-24     | 3 mixed + 1 mock contest weekly | All — company-tagged |

Cumulative target: **~250-300 problems** by placement day. Quality > quantity.
