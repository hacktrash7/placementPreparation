# The 1-Page Resume Guide for a CSE Fresher

## Hard rules

- **One page.** Two pages = recruiter skips. (Reread that.)
- **PDF only.** Never `.docx`. Name file `Firstname_Lastname_Resume.pdf`.
- **No photo, no DOB, no marital status, no Aadhaar number.** Just name + contact.
- **No "References available on request"** — that line is dead.
- **Black + white only.** Maybe one accent colour for section headers.
- **Use a clean font** — Calibri / Helvetica / Lato / Inter at 10-11pt.
- **Consistent dates format**: `Jun 2024 – Aug 2024`. Never mix.
- **Use the past tense** for bullets ("Built", "Designed", "Reduced…").
- **Quantify whenever possible** ("reduced page load by 40%", "200 SQL queries").
- **Spell-check 3 times.** A single typo can cost you the screening.

## Section order (top to bottom)

1. **Header** — Name, Phone, Email, GitHub, LinkedIn, Location (city only).
2. **Education** — Most recent first.
3. **Technical Skills** — Languages, tools, concepts. Tight.
4. **Projects** (3 max) — Most important section for freshers.
5. **Internships / Work Experience** — If any.
6. **Certifications & Coursework**
7. **Achievements** — Hackathons, ranks, papers, scholarships.
8. **Positions of Responsibility** — Club, fest, society roles.

## What every bullet should look like

```
<Strong action verb> + <what you did> + <tech used> + <impact / metric>.
```

Examples:
- **Bad**:  "Made a website using HTML."
- **Better**: "Built a personal portfolio website with 4 project pages, hosted on GitHub Pages."
- **Best**:  "Designed and shipped a 5-page portfolio website (HTML/CSS/JS) with responsive design; live on GitHub Pages, 200+ visits in first month."

## Strong action verbs (mix them up)

> Built, Designed, Implemented, Developed, Engineered, Created, Deployed, Automated, Refactored, Optimised, Analysed, Reduced, Improved, Delivered, Led, Coordinated, Mentored.

## Common mistakes to avoid

- Listing **dozens of skills** you can't defend. If "C++" is on your resume and you can't write a Stack class, that's instant disqualification.
- Generic objectives: "Seeking a challenging position…". Delete it.
- Putting **percentages without context** ("8.5 CGPA out of …"). Always show the scale.
- Project bullets that read like an essay. Stick to **2-4 bullets** per project.
- Mentioning **classroom labs as projects** ("OS lab: scheduling simulation"). Only mention if it's > 200 lines, on GitHub, and you understand it end-to-end.
- **Lying.** Service MNCs do reference checks for higher packages. Don't claim a project you didn't build.

## How to tailor for different companies

- **TCS / Infosys / Wipro / Cognizant**: emphasise **Java/Python + SQL + good academics**.
- **Tech Mahindra**: add **communication** focus — debate team, public-speaking, languages.
- **TCS Digital / Infosys DSE / Wipro Turbo**: highlight **DSA problem count + cloud / AI mini-project**.
- **Smaller startups**: emphasise **full-stack project + GitHub activity** more than CGPA.

## Format: how to actually make it

### Easy path: Microsoft Word or Google Docs

- Use the *FAANGPath simple template* or any single-column template.
- Export as PDF.

### Cleaner path: Overleaf (LaTeX)

- Use the **"Jake's Resume"** template on Overleaf. It's the de-facto fresher template, free, single-column, ATS-friendly.
- Save the .tex; rebuild the PDF every time you update.

### ATS-friendly checklist

- Single column (no fancy tables / icons).
- Plain text — no images of text.
- Standard section names (Education, Skills, Projects).
- PDF (text-based, not scanned).
- Use the **company name in keywords** where natural (Python, SQL, REST APIs, Agile…).

## Resume review checklist (do before submitting)

- [ ] Fits on one page in default zoom.
- [ ] All links work (Ctrl-click in PDF).
- [ ] No spelling/grammar errors (paste into Grammarly).
- [ ] Each project has 2-4 bullets with tech + impact.
- [ ] Skills section reflects only what you can defend in interview.
- [ ] CGPA + 10th/12th % shown.
- [ ] Phone number with country code.
- [ ] Email is a professional one, not `cooldude@gmail.com`.

Get the resume reviewed by **1 senior** who has been placed in the last 2 years before finalising.
