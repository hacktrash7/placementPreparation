# Company-Specific Interview Patterns

These are the **interview formats** (post online test) for the major recruiters. Patterns are stable year over year but verify with seniors in your college who joined recently.

## TCS Ninja / Digital

### Rounds
1. **Online Test** (already done — see `00-roadmap/placement-landscape.md`).
2. **Technical Interview** — 20-40 min.
3. **Managerial Round (MR)** — 15-30 min (Digital track usually has this).
4. **HR Round** — 10-20 min.

### Technical interview — what to expect

- 1-2 **DSA-flavoured** questions (often code on paper / Notepad / shared editor).
- Heavy **CS theory**: pick 1-2 of DBMS / OS / CN / OOP.
- "Walk me through your project" → deep questions on choices.
- Language questions on whatever you put on your resume (Java/Python/C).
- *Common opener*: "Tell me about yourself."

### Sample questions actually asked

- Difference between C and C++ / Java and Python.
- What is normalization? Explain 1NF/2NF/3NF with an example.
- Difference between process and thread.
- What is virtual memory?
- Reverse a linked list — write code.
- 3-way handshake.
- Find the second largest number in an array.
- What's the difference between OOP and procedural?

### MR/HR

- "Tell me about yourself" (1-2 min crisp story).
- "Why TCS?"
- "Are you comfortable with relocation / shift work / training in Mahindra City?"
- "What is the service agreement / bond at TCS?" — *they expect you to know.* (~2-year service agreement for Ninja/Digital.)
- "Where do you see yourself in 5 years?"
- Strengths / weaknesses, situational ("manager-disagreement", "tight deadline").

## Infosys SP / SE / Power Programmer / DSE

### Rounds
1. **Online Test** (DA Engineer / Specialist / Power Programmer have additional advanced coding section).
2. **Technical + HR combined** (~30-60 min) for SP. Separate rounds for higher tracks.

### Technical — what to expect

- 1 coding problem live (medium-level for SP, harder for Power Programmer / DSE).
- DBMS + OS + CN one-liners.
- Project deep-dive — Infosys interviewers love to drill projects.
- **Pseudocode questions** (Infosys style — flowcharts, output prediction).
- *Be ready for "Why Infosys?" and "Why should we hire you?"*.

### Sample questions actually asked

- Difference between SQL and NoSQL.
- Write a Python function to check palindrome (with comments).
- What is polymorphism? Give an example in Python.
- Explain ACID with a real-world example.
- What is the role of `static` keyword in Java?
- Tell me about the most complex bug you ever fixed (project deep-dive).

## Tech Mahindra

### Rounds
1. **Online Aptitude + Versant English + Essay + Coding** (online test).
2. **Technical Interview** — short, often phone or video.
3. **HR Interview**.

### Technical — what to expect

- *Lighter* technical than TCS / Infosys for the same offer level.
- Heavy on CS theory MCQs already in the test; interview focuses on **communication** + project + 1-2 short theory questions.
- "Walk me through your project."
- "What is the difference between TCP and UDP?" (basic CN questions are favourites).
- "Sort algorithms — which one is fastest?"
- "Explain ACID."

### HR

- The interviewer is gauging **client-readiness**. They want a polite, articulate person.
- "How will you handle an angry client?" (situational).
- "Are you comfortable with night-shift / on-site / Hyderabad relocation?"
- *Maintain eye contact, smile, take a 1-sec pause before answering.*

## Wipro Elite NLTH / Turbo

### Rounds
1. **Aptitude + Verbal + Logical + Written Communication (essay) + Coding (2 Qs)**.
2. **Technical Interview** — DSA + OOP + DBMS basics.
3. **HR Interview**.

### Technical — what to expect

- 1 coding problem live (write or pseudo-code).
- Strong focus on **OOP concepts** and **Java basics** (since Wipro trains in Java for Turbo).
- DBMS queries (write a join).
- Project Q&A.

### HR

- Standard "tell me about yourself" + "Why Wipro?" + bond info (~15-month bond).
- Often takes <15 min.

## Cognizant GenC / GenC Elevate / Elevate Next / Pro

### Rounds
1. **Online Test** (similar pattern: apt + logical + verbal + coding).
2. **Technical Interview** — DSA + project + theory.
3. **HR Interview**.

### Technical — what to expect

- Higher tracks (GenC Pro / Elevate) ask 1 medium DSA question + system design lite.
- Lower (GenC): basic coding (write code for armstrong / pattern / palindrome) + theory.
- Strong DBMS + OOP focus.
- *Confidence on basic Python/Java syntax is key.*

### HR

- "Why Cognizant?", "Future plans?", "Cognizant work culture knowledge."
- They check **clarity of communication**.

## Accenture

### Rounds
1. **Cognitive + Technical assessment + Coding + Communication test**.
2. **Technical Interview**.
3. **HR Interview**.

### Technical — what to expect

- *Less* DSA, *more* concept questions.
- DBMS + OOP + Networks rapid-fire.
- "Difference between abstract class and interface."
- "Write SQL: find 2nd highest salary."
- 1 short coding problem (palindrome, prime, etc.).

### HR

- Heavily on communication and willingness to learn.
- "Do you know about Accenture's myWizard / GenAI initiatives?" — keep one buzzword answer ready.

## Capgemini

### Rounds
1. **Online Test**: Pseudocode + Apt + English + Essay + Game-based + Coding.
2. **Technical + HR combined** (~30 min) — often the lightest interview of all majors.

### What to expect

- 1 coding (palindrome / factorial / pattern).
- Basic OOP definitions.
- "Tell me about yourself."
- "Why Capgemini?" — mention digital transformation, global presence.

## LTIMindtree / Mphasis / HCL TechBee

### Mostly similar:
1. Online test (apt + coding).
2. Single technical-HR interview.

Be ready for the standard set: project deep-dive, 1 coding, 2-3 theory questions, "Why us?".

## Universal preparation checklist (5 days before any interview)

- [ ] 60-second **self-introduction** memorised — practice in mirror 10 times.
- [ ] **"Why <company>?"** — 2-3 sentences with one real fact about the company.
- [ ] **3 strengths** with examples; **2 weaknesses** with how you're working on them.
- [ ] **3 strongest projects** explained in 90 sec each. Be ready to whiteboard the architecture.
- [ ] **CS theory rapid-fire**: 10 Qs / subject. Use [`../04-cs-core/`](../04-cs-core/) quick revision cards.
- [ ] **Last 5 LeetCode problems** revisited so you can explain solutions cleanly.
- [ ] **Resume bullets** — be ready for any follow-up on each.
- [ ] **One question to ask the interviewer** (signals interest):
      - "What does a typical first-year associate's day look like?"
      - "What technologies do new joiners get trained on these days?"
      - "How does <company> support skill upgrades like cloud / AI?"

Pin this checklist on your wall during the placement season.
